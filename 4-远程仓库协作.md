
1. git remote -v

查看当前配置的远程仓库地址

```
➜  git-demo git:(main) ✗ git remote -v
origin	git@github.com:morningcat2018/git-demo.git (fetch)
origin	git@github.com:morningcat2018/git-demo.git (push)
```

2. git remote add `<name>` `<url>`

添加一个新的远程仓库并命名（通常是 origin）

3. git fetch

从远程仓库拉取最新数据，但不会自动合并到当前分支

```
执行前
local main:   A --- B
origin/main:  A --- B --- C --- D

fetch 之后
local main:        A --- B
origin/main:       A --- B --- C --- D   (更新了！)
```

4. git pull

从远程拉取代码并自动合并到当前分支（等同于 git fetch + git merge）

---

单纯使用 git pull `可能`会额外创建一个 merge 记录,会使得搜索commit历史变得复杂,建议使用

`git pull --rebase`

或者

git config pull.rebase true

会自动加上 --rebase

5. git push

将本地提交推送到远程仓库的对应分支

git push -u origin `<branch-name>` 

推送本地分支，并建立与远程分支的跟踪关系
