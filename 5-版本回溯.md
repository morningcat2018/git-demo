1. git reset HEAD `<file>`

将文件从暂存区移回工作区

类似于 git restore --staged `<file>`

2. git reset --soft `<commit>`

回退到指定提交，保留工作区和暂存区所有改动

3. git reset --hard `<commit>`

危险！ 彻底回退，丢弃工作区和暂存区所有改动

4. git revert `<commit>`

生成一个新提交来抵消指定提交的更改（安全的公共历史回退）

---

git revert = 用“新增一个反向提交”来撤销某个历史 commit 的影响

注意：它不会删除历史 commit

```
假设历史是：

A --- B --- C --- D (HEAD)

你执行：

git revert C

Git 会做：

生成一个新的 commit E，把 C 的修改“反向应用”

结果变成：
A --- B --- C --- D --- E

👉 C 还在，只是被“抵消”了


revert 的核心特点
✅ 安全
不改历史
不会影响别人已经 pull 的代码
✅ 可追踪
有“撤销记录”
❌ 不删除 commit
commit 仍然存在


如果使用 git reset --hard C

C 之后的 commit 全没了
```
